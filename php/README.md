# php

PHP runtime tree for the module-by-module replacement described in
`docs/php-db-ui-micro-job-prompts-0351-0670.md` (Phase B: PHP Runtime
Skeleton, 0391-0440; Phase C: ANSI SQL and MariaDB Portable DB Layer,
0441-0520).

On this branch only the persistence skeleton has landed so far:

- `src/Persistence/` — `MintWiki\Persistence\ConnectionConfig` and
  `MintWiki\Persistence\PdoConnectionFactory` (0484), which build a PDO
  DSN for either PostgreSQL (`pgsql`) or MariaDB (`mysql`) from a shared
  config value object. See `docs/db-adapter-contract.md` and
  `docs/postgresql-dsn-compatibility.md` for the field-level contract this
  follows. `MintWiki\Persistence\PdoTransaction` (0485) wraps an injected
  `PDO` connection with only the `begin`/`commit`/`rollback` transaction
  contract from `docs/db-adapter-contract.md` §2.
- `tests/Persistence/` — smoke tests for the classes above, run directly
  with the `php` CLI (no phpunit).

The HTTP layer, front controller, and module namespaces from Phase B are
tracked separately and are not part of this branch's history.

`vendor/` and `composer.lock` are generated by `composer install` and are
git-ignored (`php/.gitignore`) — `composer.json` declares no packages
beyond the `php` engine constraint, so regenerating them never touches the
network.
